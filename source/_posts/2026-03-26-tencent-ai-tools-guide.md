## 二、核心案例：会议纪要自动化

这是**企业最容易付费的 AI 场景之一**。

### 传统流程（低效）

```
开会 → 手写/录音 → 会后整理 → 发纪要 → 跟进执行
```

| 痛点 | 说明 |
|------|------|
| 记录不完整 | 记忆偏差导致信息遗漏 |
| 整理耗时 | 30-60 分钟整理会议纪要 |
| 容易遗漏 | 关键决策被淹没在讨论中 |

### 腾讯 AI 方案（升级版）

**工具组合：**
- 腾讯会议（录音/转写）
- 腾讯元宝（总结分析）

### 自动化流程

```
会议进行（腾讯会议录音/转写）
        ↓
导出会议内容（文本）
        ↓
丢给元宝（结构化处理）
        ↓
自动生成：
  - 会议纪要
  - 决策点
  - 待办事项（Action List）
        ↓
同步到腾讯文档 / Workbuddy
```

### 元宝提示词模板（直接可用）

```
请根据以下会议内容，输出结构化纪要：

1. 会议主题
2. 核心讨论点（分点列出）
3. 已达成决策
4. 待办事项（责任人 + 截止时间）
5. 风险与问题

要求：
- 简洁清晰
- 使用项目符号
- 适合直接发给团队
```

### 输出效果（示例）

```
【会议主题】
Q2产品增长策略讨论

【核心讨论】
- 用户增长停滞原因分析
- 新渠道投放策略

【决策】
- 增加短视频投放预算30%
- 启动AI客服试点

【待办事项】
- 张三：制定投放方案（3月30日）
- 李四：测试AI客服（4月5日）

【风险】
- ROI不确定
- 数据监控不足
```

### 再升级（关键！）

把纪要接入 **Workbuddy**，自动触发：

- 创建任务
- 分配责任人
- 跟踪进度
- 到期提醒

## 三、完整闭环（真正的腾讯打法）

```
腾讯会议（输入）
   ↓
元宝（理解 + 总结）
   ↓
腾讯文档（沉淀）
   ↓
Workbuddy（执行）
```

> **本质：会议 ≠ 讨论，而是"自动生成行动系统"**
